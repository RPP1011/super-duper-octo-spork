### US-007: Implement explicit region target picker workflow
Status: in_progress
Depends on: US-006

Description:
As a player, I want map-click region target assignment for delegated orders so order intent is precise and discoverable.

Acceptance Criteria:
- [ ] Region target picker supports enter-select-confirm-cancel flow for delegated order assignment.
- [ ] Chosen target region is displayed in party panel and used by delegated movement.
- [ ] Picker state is visually distinct from normal selection interactions.
- [ ] Example: assign Patrol to Region-C through picker and observe party target state update to Region-C.
- [ ] Negative case: confirming without a valid region selection is rejected and preserves previous target.
- [ ] `cargo test` passes including interaction tests for picker state transitions and target persistence.
