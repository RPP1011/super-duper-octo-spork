### US-003: Isolate menu subtitle/status text from in-session notices
Status: in_progress
Depends on: US-002

Description:
As a player, I want start-menu messaging separated from runtime notices so the menu context remains clear and stable.

Acceptance Criteria:
- [ ] Start-menu subtitle source is decoupled from hub/campaign notice channels.
- [ ] In-session action notices no longer appear on start menu after returning to it.
- [ ] Example: completing a mission action notice, then returning to menu, shows menu-appropriate subtitle copy only.
- [ ] Negative case: stale hub notice text must not appear as start-menu subtitle under any menu re-entry path.
- [ ] `cargo test` passes with coverage for state-source isolation between menu and runtime notices.
