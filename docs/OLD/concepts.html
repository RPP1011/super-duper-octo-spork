What is Attention?

Attention is a way for a neural network to selectively focus on the most relevant parts of its input. Instead of treating all information equally, attention asks: “given what I’m looking for, what should I pay attention to?”

The mechanism has three components, best understood through analogy:

The key insight: Query and Key are in the same space, so their dot product measures compatibility. A high dot product means “this Key matches what the Query is looking for.” After softmax normalization, the resulting weights select which Values to read.

In our game AI, this appears in three places:

  • Entity encoder — self-attention: entities query each other (“who on the battlefield matters to me?”)
  • Ability integration — cross-attention: abilities query entities (“given what this ability does, which entities are relevant?”)
  • Target selection — pointer attention: the combat head queries entities (“who should I target?”)

Self-Attention

Self-attention lets each entity in the game "look at" every other entity to decide what matters. The entity encoder uses 4-head self-attention so that a healer can notice a wounded ally, or a damage-dealer can spot a vulnerable enemy — all learned, not hand-coded.

Attention(Q, K, V) = softmax(QKT / √d) · V

Cross-Attention

Cross-attention is like self-attention, but the query comes from a different source. Here, a pretrained ability embedding (e.g. "Fireball") generates the query, while entity tokens produce the keys and values. This lets each ability "ask" the battlefield: which entities am I most relevant to?

Q = Wq · ability_emb   K = Wk · entity_tokens   out = softmax(QKT / √32) · V

Logits & Softmax

The model's raw output scores (logits) are arbitrary real numbers — they could be negative, large, or close together. Softmax converts them into a probability distribution that sums to 1.0. The temperature parameter controls how "sharp" or "flat" the distribution is: low temperature concentrates probability on the top action, high temperature spreads it out.

P(i) = exp(zi / τ) / Σj exp(zj / τ)

Masked Language Modeling (MLM)

MLM is a pretraining technique where we randomly hide (mask) some tokens in the input and train the model to predict them. By learning to fill in the blanks of ability DSL text, the transformer learns what each token means in context. The [CLS] token's output becomes a 128-dimensional summary embedding that captures the ability's full behavior.

Loss = − Σmasked log P(tokentrue | context)

Scaled Dot-Product Attention

The fundamental computation behind all attention mechanisms. A query vector is compared against multiple key vectors using dot products, scaled down by √d to prevent the softmax from saturating, then used to take a weighted average of value vectors. In our pointer heads, this is how the model selects which entity to target.

scorei = Q · KiT / √dk    αi = softmax(score)i    out = Σi αi Vi
