### US-002: Enforce start-menu action hierarchy and continue semantics
Status: in_progress
Depends on: US-001

Description:
As a player, I want a clear start menu where New Campaign is primary and Continue resumes correctly so onboarding is predictable.

Acceptance Criteria:
- [ ] Start menu prioritizes New Campaign visually and interaction-wise over other actions.
- [ ] Continue Campaign resumes latest compatible save/autosave into correct campaign context instead of unrelated screens.
- [ ] Saves diagnostics remain collapsed by default and do not dominate first-run readability.
- [ ] Example: with a valid autosave, selecting Continue enters saved overworld context in one action.
- [ ] Negative case: with no compatible saves, Continue shows explicit failure status and remains on start menu.
- [ ] `cargo test` passes including tests for continue-selection behavior with and without compatible saves.
