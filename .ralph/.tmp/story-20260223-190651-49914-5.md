### US-006: Deliver party list control UX with command handoff
Status: in_progress
Depends on: US-001, US-005

Description:
As a player, I want clear party selection and command handoff controls so I can manage multiple parties intentionally.

Acceptance Criteria:
- [ ] Overworld panel lists all parties with clear markers for selected and directly controlled parties.
- [ ] Take Command action transfers direct control to eligible delegated party and updates focus context.
- [ ] Party order and notice information is visible and updates after actions.
- [ ] Example: selecting delegated Party-B then Take Command makes Party-B controlled and previously controlled party delegated.
- [ ] Negative case: Take Command on invalid/ineligible party is blocked with explicit reason and no state mutation.
- [ ] `cargo test` passes with tests for command handoff invariants and UI state bindings.
