### US-001: Complete party state foundation and persistence
Status: in_progress
Depends on: None

Description:
As a developer, I want campaign party state fully defined and persisted so multi-party gameplay survives save/load without ambiguity.

Acceptance Criteria:
- [ ] Campaign party state includes selected party, leader mapping, order kind, current/target region, and notice fields.
- [ ] Save/load serializes and restores all party fields required by the model without lossy fallback.
- [ ] Example: after saving with one delegated party on Patrol targeting Region-B, loading restores Patrol + Region-B target exactly.
- [ ] Negative case: loading a save missing party fields yields a compatibility failure status and avoids partial corrupted runtime state.
- [ ] `cargo test` passes including tests that cover party state round-trip serialization.
