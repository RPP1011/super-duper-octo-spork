### US-008: Add camera focus interpolation for party swaps
Status: in_progress
Depends on: US-006

Description:
As a player, I want smooth camera focus transitions when switching controlled parties so orientation remains understandable.

Acceptance Criteria:
- [ ] Party focus action interpolates camera toward selected/controlled party rather than snapping abruptly.
- [ ] Focus transition locks conflicting camera commands until interpolation completion or safe interrupt path.
- [ ] UI reflects transition state to avoid ambiguous input outcomes.
- [ ] Example: hand off control to Party-B triggers smooth move and finishes with Party-B centered and selected region synced.
- [ ] Negative case: repeated rapid focus actions do not leave camera in undefined state or desync selected party.
- [ ] `cargo test` passes with deterministic tests for interpolation state machine behavior.
