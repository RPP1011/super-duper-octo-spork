### US-009: Implement overworld-to-region transition payload
Status: in_progress
Depends on: US-007, US-008

Description:
As a player, I want explicit region entry from overworld so progression into regional gameplay is intentional and deterministic.

Acceptance Criteria:
- [ ] Overworld exposes explicit region-entry action tied to selected region context.
- [ ] Transition payload includes region id, faction context, and deterministic seed data needed by region layer.
- [ ] During transition, interaction is state-locked to prevent dual-layer input conflicts.
- [ ] Example: entering Region-D loads region scene with Region-D id and expected campaign seed-derived context.
- [ ] Negative case: entering region with missing payload fields fails gracefully and returns to overworld with clear status.
- [ ] `cargo test` passes with payload contract tests and transition-guard tests.
