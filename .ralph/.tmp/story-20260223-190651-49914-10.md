### US-011: Persist layered campaign progression and migration guards
Status: in_progress
Depends on: US-010

Description:
As a returning player, I want saves to resume at the correct campaign layer with compatibility protection so progress remains stable across updates.

Acceptance Criteria:
- [ ] Save data includes character creation choices, current layer, region/local context, party state, and intro completion flag.
- [ ] Load flow restores into menu, overworld, region, or local layer according to saved progression marker.
- [ ] Migration checks detect incompatible versions and provide non-destructive fallback messaging.
- [ ] Example: save during region layer then load resumes same region context with preserved selected party.
- [ ] Negative case: incompatible version save is rejected with explicit compatibility status and no partial load side effects.
- [ ] `cargo test` passes with migration and layer-resume coverage.
